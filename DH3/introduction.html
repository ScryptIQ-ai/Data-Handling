    Introduction DH3

    Welcome to Data Handling 3, where we will begin your journey on conceptualising images as multi-dimensional arrays of numbers. Once transformed into NumPy arrays we can treat images like any other mathematical array of numbers, slicing them, filtering them, and applying augmentations.

    In this lesson we will begin with simple grey scale images, to colour images (both low resolution and high resolution), and finally with 3-dimensional images, such as an MRI scan. Hopefully, by the end of it you will be confident to tackle any image you have at your own disposal.

    Learning Overview

    Concept Synergy Syntax & Functions Objective / Application Common Pitfalls / Notes
    Images as Arrays Builds on DH1 NumPy arrays; extends to 2-D and 3-D spatial data imread(), .shape, type() Understand that images are N-dimensional arrays of pixel/voxel intensity values Confusing pixel coordinates with array indexing; forgetting images use (height, width) not (x, y)
    Reading & Plotting Images Extends DH1 plotting with matplotlib; applies array slicing from DH1 imread(path), ax.imshow(img, cmap='gray') Load image files (jpg, png, tif) and visualize them as arrays Not setting appropriate colormap; imread returns different types for different formats
    Pixel Intensity Values Uses NumPy array indexing from DH1; foundation for quantitative image analysis img[row, col], img[:50, :70] Access and interpret individual pixel values (0-255 for 8-bit images) Confusing row/column order; not understanding that lower values = darker in grayscale
    RGB Color Images Extends 2-D arrays to 3-D with channel dimension; builds on array slicing from DH1 img[:, :, 0], img[:, :, 1], img[:, :, 2] Work with 3-channel color images where each pixel has R, G, B intensity values Forgetting third dimension is channels not depth; mixing up channel order (0=red, 1=green, 2=blue)
    Image Histograms Applies DH1 histogram plotting to pixel intensity distributions ax.hist(img.flatten(), bins=50) Visualize distribution of pixel intensities to inform thresholding decisions Not using enough bins; forgetting to flatten multi-dimensional arrays before plotting
    Boolean Masks Builds on DH1 conditional filtering; applies logical operations to arrays mask = img < threshold, mask = img > threshold Create binary True/False arrays based on intensity thresholds Confusing direction of inequality; not understanding True=1, False=0 in operations
    Applying Masks Extends DH1 filtering operations; element-wise multiplication from NumPy img_masked = img * mask Filter images to isolate regions of interest based on intensity criteria Forgetting multiplication zeros out False regions; not preserving original intensity values
    Mask Inversion Logical NOT operation on boolean arrays ~mask, img[~mask, :] = background Flip boolean mask values (True↔False) to select opposite regions Forgetting ~ inverts the selection; not understanding when to invert for assignment operations
    Large Image Handling Memory management considerations; extends DH1 efficiency concepts Image.MAX_IMAGE_PIXELS, .resize(), extract_patches_2d() Process high-resolution images that exceed memory limits Running out of memory; not recognizing decompression bomb warnings
    PIL/Pillow Library Alternative to matplotlib for image I/O; specialized image processing tools Image.open(path), img.resize((w, h)), array(img_pil) Load, resize, and manipulate images with PIL before converting to NumPy Confusing PIL Image objects with NumPy arrays; forgetting to convert back to array
    Remote Image Loading Combines URL handling with image I/O; extends file operations from PF4 urlopen(url), BytesIO(), imread(remote_image) Load images from remote URLs instead of local files SSL certificate issues; slow downloads; not handling network errors
    Voxels 3-D extension of pixels; volume elements in 3-D images img_data[x, y, z], img_data.shape (returns 3-D dimensions) Access and manipulate 3-D volume elements in medical images Confusing voxel coordinates across different anatomical planes
    3-D Image Slicing Applies DH1 slicing to three dimensions; visualize planes through volumes img[:, :, z] (transverse), img[:, y, :] (coronal), img[x, :, :] (sagittal) Extract 2-D slices from 3-D volumes across different anatomical planes Forgetting to use .T for proper orientation; confusing which dimension to fix
