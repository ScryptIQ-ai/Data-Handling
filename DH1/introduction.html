    Introduction DH1

    Welcome to the data Data Handling module, where we will introduce with several methods to handle an suite of data types that are hopefully relevant to your discipline. For researchers, everything comes down to data, so knowing how to properly manipulate, augment, and analyse any data in Python will stand you in good stead for the future. Unlocking a myriad of further packages in Python for actions such as beautiful plotting to niche Python packages for analysis in your domain.

    We'll begin by working with structured data using one of Python's most powerful data analysis libraries: pandas. You'll explore the pandas DataFrame—the core data structure for tabular data—learning essential operations like indexing, slicing, filtering, and handling missing values. You'll also create insightful visualisations using both Pandas' built-in plotting capabilities and Matplotlib.

    Learning Overview

    Concept Synergy Syntax & Functions Objective / Application Common Pitfalls / Notes
    DataFrames Central structure for all Pandas operations; contains rows, columns, and index df = read_csv(...), df.head(), df.shape Store and manipulate tabular data with labeled rows and columns Confusing index with data; forgetting that metadata (index/columns) isn't part of shape
    Viewing Data Essential for initial data exploration and verification df, df.head(), df.head(n), print(df) Display DataFrame contents to understand structure and values Not checking data after loading; assuming data loaded correctly without verification
    DataFrame Properties Provides metadata about DataFrame structure df.shape, df.columns, df.dtypes, len(df) Understand dimensions, column names, and data types Confusing properties (no parentheses) with methods (with parentheses)
    Indexing & Slicing Connects to Python list slicing; essential for data access df['column'], df['column'][0:3], df.loc[rows, columns] Extract specific data subsets from DataFrames Mixing .loc (label-based, inclusive) with list slicing (semi-inclusive); nested bracket confusion
    Label-based Indexing (.loc) Preferred modern approach for accessing data by labels df.loc[:, 'column'], df.loc[0:2, 'column'], df.loc[[1,3], ['col1', 'col2']] Access rows and columns using explicit labels .loc includes upper boundary unlike standard slicing; forgetting colon for "all rows/columns"
    Missing Values Critical for data quality; affects statistical calculations and visualizations df.isnull(), df.isnull().any(), df.fillna(value) Identify and handle NaN values in datasets Not handling NaN before analysis; choosing inappropriate replacement values
    Summary Statistics Builds on mathematical operators; provides data overview df.describe(), df.mean(), df.std(), df.sum() Generate statistical summaries to understand data distributions Misinterpreting percentiles; not recognizing that .describe() returns a DataFrame
    Creating New Columns Applies mathematical and logical operators to DataFrame columns df['new_col'] = expression, df['col1'] + df['col2'], df['col'] > value Derive new data from existing columns through calculations or conditions Column length mismatch; including spaces in column names
    Filtering Data Combines Boolean logic with indexing df[df['column'] > value], df[condition] Select rows that meet specific criteria Creating filters but not applying them; complex multi-condition logic errors
    Pandas Visualization Quick plotting built on Matplotlib; useful for rapid exploration df.boxplot(), df.boxplot(column='name') Create simple plots directly from DataFrame Limited customization compared to Matplotlib; forgetting to handle NaN values first
    Matplotlib Basics Foundation for advanced visualization; more control than Pandas plotting from matplotlib.pyplot import subplots, show, fig, ax = subplots() Create customizable, publication-quality visualizations Forgetting to import; confusion between fig and ax objects
